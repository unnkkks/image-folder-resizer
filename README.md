# image-folder-resizer

Здесь находится приложение для изменения размеров изображений в папке.  
Основная идея проекта — взять все изображения из входной директории, изменить их размер по выбранному режиму работы с пропорциями и сохранить результат в отдельную папку.

`image-folder-resizer` — это приложение на Python для пакетного изменения размеров изображений в папке. Программа обрабатывает все поддерживаемые изображения из входной директории и сохраняет результат в выходную директорию.

В проекте используется структура `src/`, тесты `pytest`, проверка типов `mypy`, форматирование и линтинг `ruff`, CLI на `click`, документация `Sphinx`, задачи запуска через `Poe the Poet`, управление зависимостями через `uv`, а также CI/CD через GitHub Actions.
=======
# Теоретические сведения об изменении размеров изображений

При изменении размера изображения важно учитывать не только новую ширину и высоту, но и **соотношение сторон** — Aspect Ratio.

Соотношение сторон показывает, как ширина изображения относится к его высоте:

$$
aspect = \frac{width}{height}
$$

Например, изображение `1920x1080` имеет соотношение сторон `16:9`, а изображение `1000x1000` — `1:1`.

Если просто изменить изображение до нового размера без учета пропорций, оно может растянуться или сжаться неестественно. Поэтому в проекте реализовано несколько режимов изменения размера:

1. **Stretch** — растяжение или сжатие до точного размера  
2. **Fit** — вписывание изображения с сохранением пропорций  
3. **Fill** — заполнение области с сохранением пропорций и обрезкой  
4. **Pad** — вписывание с сохранением пропорций и добавлением фона  

## 1. Stretch

Метод `stretch` изменяет изображение строго до размера, который указал пользователь.

Если исходное изображение имеет размер:

$$
W_o \times H_o
$$

а пользователь указал новый размер:

$$
W_t \times H_t
$$

то изображение просто преобразуется к размеру:

$$
W_t \times H_t
$$

В коде это соответствует обычному вызову resize:

```python
image.resize((target_width, target_height))
```

**Особенности и применение**

- Итоговое изображение всегда имеет ровно тот размер, который указал пользователь.
- Пропорции не сохраняются.
- Изображение может исказиться, если новый размер сильно отличается от исходного.
- Подходит для случаев, когда важен именно фиксированный размер, а не внешний вид.

Пример:  
изображение `1200x800` при изменении до `512x512` станет квадратным, но объекты на нем могут быть растянуты.

## 2. Fit

Метод `fit` сохраняет пропорции изображения и вписывает его внутрь заданной области.

Для этого считается коэффициент масштабирования:

$$
scale = \min\left(\frac{W_t}{W_o}, \frac{H_t}{H_o}\right)
$$

где:

- $W_o$ — исходная ширина;
- $H_o$ — исходная высота;
- $W_t$ — требуемая ширина;
- $H_t$ — требуемая высота.

Новый размер вычисляется так:

$$
W_n = W_o \cdot scale
$$

$$
H_n = H_o \cdot scale
$$

Используется именно минимум, потому что изображение должно полностью поместиться внутрь заданной области.

**Особенности и применение**

- Пропорции сохраняются.
- Изображение не обрезается.
- Итоговый размер может быть меньше заданного по одной из сторон.
- Хорошо подходит для предварительного просмотра изображений и безопасного уменьшения картинок.

Пример:  
если изображение `1200x800` вписать в область `512x512`, оно станет примерно `512x341`.

## 3. Fill

Метод `fill` тоже сохраняет пропорции, но работает иначе: изображение должно полностью заполнить заданную область.

Коэффициент масштабирования считается так:

$$
scale = \max\left(\frac{W_t}{W_o}, \frac{H_t}{H_o}\right)
$$

Новый размер:

$$
W_n = W_o \cdot scale
$$

$$
H_n = H_o \cdot scale
$$

После масштабирования изображение может стать больше нужной области по одной из сторон. Лишняя часть обрезается по центру.

Координаты для центральной обрезки можно описать так:

$$
left = \frac{W_n - W_t}{2}
$$

$$
top = \frac{H_n - H_t}{2}
$$

$$
right = left + W_t
$$

$$
bottom = top + H_t
$$

**Особенности и применение**

- Пропорции сохраняются.
- Итоговое изображение имеет ровно заданный размер.
- Часть изображения может быть обрезана.
- Подходит для аватарок, обложек, карточек товаров и превью.

Пример:  
если изображение `1200x800` привести к `512x512` через `fill`, оно сначала масштабируется так, чтобы закрыть весь квадрат, а затем лишние края обрезаются.

## 4. Pad

Метод `pad` сохраняет пропорции и добавляет фон до нужного размера.

Сначала изображение уменьшается или увеличивается по принципу `fit`:

$$
scale = \min\left(\frac{W_t}{W_o}, \frac{H_t}{H_o}\right)
$$

Затем создается новое пустое изображение размера:

$$
W_t \times H_t
$$

После этого уменьшенное изображение вставляется в центр. Координаты вставки:

$$
x = \frac{W_t - W_n}{2}
$$

$$
y = \frac{H_t - H_n}{2}
$$

Пустые области заполняются выбранным цветом фона.

**Особенности и применение**

- Пропорции сохраняются.
- Изображение не обрезается.
- Итоговый размер всегда равен заданному.
- По краям могут появиться поля.
- Хорошо подходит для подготовки изображений одинакового размера без потери содержимого.

Пример:  
изображение `1200x800` при приведении к `512x512` станет `512x341`, а сверху и снизу появятся поля.

## Сравнительная таблица режимов

| Режим | Сохраняет пропорции | Итоговый размер точный | Может обрезать изображение | Может искажать изображение | Где удобно использовать |
|------|----------------------|-------------------------|-----------------------------|-----------------------------|--------------------------|
| **stretch** | Нет | Да | Нет | Да | Когда нужен строго фиксированный размер |
| **fit** | Да | Нет | Нет | Нет | Превью, безопасное уменьшение |
| **fill** | Да | Да | Да | Нет | Аватарки, карточки, обложки |
| **pad** | Да | Да | Нет | Нет | Единый размер без потери содержимого |

# Как работает программа

Работа программы строится вокруг обработки папки с изображениями.

Общий алгоритм:

1. Пользователь запускает программу через командную строку.
2. Указывает входную папку, выходную папку, ширину, высоту и режим обработки.
3. Программа проверяет параметры.
4. Программа ищет изображения во входной папке.
5. Для каждого изображения:
   - открывает файл;
   - приводит изображение к подходящему цветовому режиму;
   - применяет выбранный режим изменения размера;
   - сохраняет результат в выходную папку.
6. После завершения выводится информация о результате обработки.

Поддерживаются изображения с расширениями:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`

# Основные модули проекта

## `aspect.py`

В этом файле находятся функции, связанные с расчетом размеров и координат.

Основные задачи модуля:

- вычислить размер для режима `fit`;
- вычислить размер для режима `fill`;
- вычислить координаты центральной обрезки;
- вычислить координаты вставки изображения по центру.

Этот модуль не занимается чтением файлов. Он отвечает только за математику.

## `image_ops.py`

В этом файле находится основная логика работы с изображениями.

Здесь реализовано:

- открытие изображения;
- обработка ошибок при открытии;
- преобразование изображения в подходящий режим цвета;
- изменение размера изображения;
- применение режимов `stretch`, `fit`, `fill`, `pad`.

Именно этот модуль отвечает за то, как будет изменено конкретное изображение.

## `io.py`

Этот файл отвечает за работу с файловой системой.

Он используется для того, чтобы:

- найти изображения в папке;
- проверить расширение файла;
- создать выходную директорию;
- сохранить результат;
- обработать вложенные папки при рекурсивном запуске.

## `cli.py`

В этом файле описан интерфейс командной строки.

Для CLI используется библиотека `click`.  
Она позволяет удобно задавать параметры запуска:

- входная папка;
- выходная папка;
- ширина;
- высота;
- режим обработки;
- качество изображения;
- формат сохранения;
- цвет фона;
- рекурсивный режим.

## `config.py`

Файл с настройками и вспомогательными функциями конфигурации.

## `exceptions.py`

Файл с пользовательскими исключениями проекта.

# Установка и запуск

Проект использует `uv`, поэтому сначала нужно установить зависимости:

```bash
uv sync
```

Пример запуска:

```bash
uv run resizer input output --width 512 --height 512 --mode stretch
```

Где:

- `input` — папка с исходными изображениями;
- `output` — папка для результата;
- `--width` — новая ширина;
- `--height` — новая высота;
- `--mode` — режим изменения размера.

Пример запуска с сохранением пропорций:

```bash
uv run resizer input output --width 512 --height 512 --mode fit
```

Пример запуска с заполнением области:

```bash
uv run resizer input output --width 512 --height 512 --mode fill
```

Пример запуска с фоном:

```bash
uv run resizer input output --width 512 --height 512 --mode pad --background 255,255,255
```

Пример запуска с рекурсивной обработкой:

```bash
uv run resizer input output --width 800 --height 600 --mode fit --recursive
```

# Параметры CLI

| Параметр | Описание |
|---------|----------|
| `input_dir` | входная папка с изображениями |
| `output_dir` | папка для сохранения результата |
| `--width` | ширина итогового изображения |
| `--height` | высота итогового изображения |
| `--mode` | режим обработки: `stretch`, `fit`, `fill`, `pad` |
| `--recursive` | обработка вложенных папок |
| `--overwrite` | перезапись существующих файлов |
| `--quality` | качество сохранения изображения |
| `--format` | принудительный формат: `jpeg`, `png`, `webp` |
| `--background` | цвет фона для режима `pad` |

# Пример работы

Пусть есть папка:

```text
input/
├── photo1.jpg
├── photo2.png
└── avatar.webp
```

Запуск:

```bash
uv run resizer input output --width 512 --height 512 --mode pad
```

После выполнения появится папка:

```text
output/
├── photo1.jpg
├── photo2.png
└── avatar.webp
```

Все изображения будут приведены к размеру `512x512`.  
При этом пропорции сохранятся, а недостающие области будут заполнены фоном.

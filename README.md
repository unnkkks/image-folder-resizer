# image-folder-resizer

[![Tests](https://github.com/unnkkks/image-folder-resizer/actions/workflows/ci.yml/badge.svg)](https://github.com/unnkkks/image-folder-resizer/actions/workflows/ci.yml)
[![Coverage](https://img.shields.io/badge/coverage-90%25%2B-brightgreen)](./coverage.xml)

`image-folder-resizer` — это приложение на Python для пакетного изменения размеров изображений в папке. Программа обрабатывает все поддерживаемые изображения из входной директории и сохраняет результат в выходную директорию.

В проекте используется структура `src/`, тесты `pytest`, проверка типов `mypy`, форматирование и линтинг `ruff`, CLI на `click`, документация `Sphinx`, задачи запуска через `Poe the Poet`, управление зависимостями через `uv`, а также CI/CD через GitHub Actions.

## Возможности

Программа умеет:

- обрабатывать все изображения в указанной папке;
- изменять размер изображений до заданной ширины и высоты;
- поддерживать разные стратегии работы с пропорциями изображения;
- сохранять результат в отдельную папку;
- рекурсивно обходить вложенные директории;
- перезаписывать или не перезаписывать уже существующие файлы;
- задавать качество сохранения;
- принудительно выбирать формат выходного файла: `jpeg`, `png`, `webp`;
- использовать цвет фона для режима с отступами;
- запускаться как CLI-команда `resizer`.

## Основная идея проекта

Пользователь указывает входную папку, выходную папку, требуемую ширину, высоту и режим изменения размера. После этого программа находит изображения, открывает их через библиотеку Pillow, применяет выбранный алгоритм изменения размера и сохраняет обработанные файлы.

Общий сценарий работы:

1. Пользователь запускает команду `resizer`.
2. CLI получает параметры: путь к папке, размер, режим, формат, качество и другие настройки.
3. Программа проверяет корректность входных данных.
4. Из входной папки собирается список поддерживаемых изображений.
5. Каждое изображение открывается и приводится к подходящему цветовому режиму.
6. К изображению применяется выбранный алгоритм изменения размера.
7. Результат сохраняется в выходную папку.
8. В консоль выводится информация о количестве обработанных файлов.

## Поддерживаемые режимы изменения размера

В проекте реализовано несколько режимов работы с Aspect Ratio, то есть с соотношением сторон изображения.

### `stretch`

Режим `stretch` изменяет изображение строго до указанного размера.

Например, если исходное изображение имеет размер `1200x800`, а пользователь указал `512x512`, то программа просто растянет или сожмет изображение до `512x512`.

Особенности режима:

- итоговый размер всегда точно совпадает с заданным;
- пропорции изображения не сохраняются;
- изображение может визуально исказиться.

Алгоритм:

```text
new_image = image.resize((target_width, target_height))
```

Этот режим подходит, когда важен строго заданный размер, а сохранение пропорций не критично.

### `fit`

Режим `fit` сохраняет пропорции изображения и вписывает его в заданную область.

Например, если нужно поместить изображение в область `512x512`, программа вычисляет такой масштаб, чтобы изображение полностью помещалось внутри этой области.

Особенности режима:

- пропорции сохраняются;
- изображение не обрезается;
- итоговое изображение может быть меньше заданного размера по одной из сторон.

Алгоритм:

```text
scale = min(target_width / original_width, target_height / original_height)

new_width = original_width * scale
new_height = original_height * scale
```

Используется минимальный коэффициент масштабирования, чтобы изображение целиком поместилось в целевой размер.

### `fill`

Режим `fill` сохраняет пропорции, но заполняет всю заданную область.

Если после масштабирования часть изображения выходит за границы целевого размера, лишние края обрезаются по центру.

Особенности режима:

- пропорции сохраняются;
- итоговый размер точно совпадает с заданным;
- часть изображения может быть обрезана;
- удобно для аватарок, карточек, превью и обложек.

Алгоритм:

```text
scale = max(target_width / original_width, target_height / original_height)

resized_width = original_width * scale
resized_height = original_height * scale

left = (resized_width - target_width) / 2
top = (resized_height - target_height) / 2

crop center area
```

В отличие от `fit`, здесь используется максимальный коэффициент масштабирования, чтобы изображение полностью закрыло нужную область.

### `pad`

Режим `pad` сохраняет пропорции и добавляет фон до нужного размера.

Сначала изображение уменьшается или увеличивается так, чтобы полностью поместиться внутрь заданной области. Затем создается новое изображение нужного размера с фоном, и исходное изображение вставляется по центру.

Особенности режима:

- пропорции сохраняются;
- изображение не обрезается;
- итоговый размер точно совпадает с заданным;
- пустые области заполняются выбранным цветом.

Алгоритм:

```text
scale = min(target_width / original_width, target_height / original_height)

resized_width = original_width * scale
resized_height = original_height * scale

x = (target_width - resized_width) / 2
y = (target_height - resized_height) / 2

create background image
paste resized image to center
```

Этот режим полезен, если нужно получить изображения одинакового размера без обрезки содержимого.

## Установка

Сначала нужно установить `uv`.

Для Linux, macOS или WSL:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После этого нужно перейти в папку проекта:

```bash
cd image-folder-resizer
```

Установить зависимости:

```bash
uv sync
```

## Запуск программы

Базовый запуск:

```bash
uv run resizer input output --width 512 --height 512 --mode stretch
```

Где:

- `input` — папка с исходными изображениями;
- `output` — папка, куда будут сохранены обработанные изображения;
- `--width` — нужная ширина;
- `--height` — нужная высота;
- `--mode` — режим изменения размера.

Пример запуска с сохранением пропорций:

```bash
uv run resizer input output --width 512 --height 512 --mode fit
```

Пример запуска с заполнением области и обрезкой:

```bash
uv run resizer input output --width 512 --height 512 --mode fill
```

Пример запуска с добавлением фона:

```bash
uv run resizer input output --width 512 --height 512 --mode pad --background 255,255,255
```

Пример запуска с рекурсивной обработкой вложенных папок:

```bash
uv run resizer input output --width 800 --height 600 --mode fit --recursive
```

Пример запуска с принудительным форматом:

```bash
uv run resizer input output --width 512 --height 512 --mode fill --format webp
```

Пример запуска с качеством JPEG или WebP:

```bash
uv run resizer input output --width 512 --height 512 --mode fill --quality 90
```

## CLI-параметры

Основные параметры команды:

```text
input_dir       входная директория с изображениями
output_dir      директория для сохранения результата
--width         ширина итогового изображения
--height        высота итогового изображения
--mode          режим изменения размера: stretch, fit, fill, pad
--recursive     обработка вложенных директорий
--overwrite     перезапись уже существующих файлов
--quality       качество сохранения изображения
--format        принудительный формат вывода: jpeg, png, webp
--background    цвет фона для режима pad
```

## Описание основных модулей

### `aspect.py`

Модуль отвечает за математические вычисления размеров.

В нем находятся функции, которые:

- вычисляют размер для режима `fit`;
- вычисляют размер для режима `fill`;
- находят координаты центральной обрезки;
- находят позицию вставки изображения по центру.

Этот модуль не работает напрямую с файлами, а только считает размеры и координаты.

### `image_ops.py`

Модуль отвечает за операции с изображениями.

В нем реализованы:

- открытие изображения;
- приведение изображения к подходящему цветовому режиму;
- изменение размера изображения;
- применение режимов `stretch`, `fit`, `fill`, `pad`.

Именно здесь находится основная логика обработки изображений.

### `io.py`

Модуль отвечает за работу с файловой системой.

Он:

- ищет изображения во входной папке;
- проверяет поддерживаемые расширения;
- создает выходные директории;
- сохраняет обработанные изображения;
- поддерживает рекурсивный обход папок.

### `cli.py`

Модуль содержит command-line interface на Click.

Он принимает параметры пользователя из терминала и передает их в основную логику программы.

### `config.py`

Модуль содержит настройки и вспомогательные функции конфигурации.

### `exceptions.py`

Модуль содержит пользовательские исключения проекта.

## Поддерживаемые форматы

Программа поддерживает основные форматы изображений:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`

Выходной формат можно оставить исходным или указать явно через параметр `--format`.

## Пример работы

Допустим, есть папка `input`:

```text
input/
├── photo1.jpg
├── photo2.png
└── avatar.webp
```

Команда:

```bash
uv run resizer input output --width 512 --height 512 --mode pad
```

После выполнения появится папка `output`:

```text
output/
├── photo1.jpg
├── photo2.png
└── avatar.webp
```

Все изображения будут приведены к размеру `512x512`, при этом пропорции сохранятся, а недостающие области будут заполнены фоном.

## Производительность

Для анализа производительности используется скрипт:

```bash
uv run python scripts/profile_resize.py
```

Он создает тестовые изображения, запускает обработку и измеряет время выполнения для разных объемов данных.

Результаты используются для построения графика производительности и описания в LaTeX-отчете.

Основной bottleneck проекта — операции открытия, декодирования, изменения размера и сохранения изображений. Это связано с тем, что обработка изображений требует чтения файлов с диска и вычислений над большим количеством пикселей.

Возможные способы ускорения:

- параллельная обработка изображений;
- уменьшение количества операций чтения и записи;
- выбор более быстрого формата сохранения;
- настройка качества изображения;
- кэширование промежуточных данных;
- обработка изображений батчами.
